모바일 UX

구매의 80%는 모바일 — 모바일 퍼스트 상세페이지 설계법

박서연2025.02.036분 읽기

2024년 기준, 국내 온라인 쇼핑의 80% 이상이 모바일에서 이루어집니다. 그런데 아직도 데스크톱에서 상세페이지를 만들고 모바일 확인을 안 하는 셀러가 많습니다.

모바일에서 상세페이지가 망가지는 이유

1. 이미지 해상도 문제

데스크톱용 고해상도 이미지가 모바일에서 로딩이 느려져 이탈률이 높아집니다. 모바일 환경을 기준으로 이미지를 2MB 이하로 최적화하세요.

2. 글자 크기 문제

이미지 안에 텍스트를 넣은 상세페이지는 모바일에서 글자가 너무 작아져 읽을 수 없게 됩니다. 핵심 텍스트는 이미지 밖(HTML 텍스트)으로 빼세요.

3. 가로 스크롤 문제

데스크톱 기준으로 폭이 넓은 이미지를 넣으면 모바일에서 가로 스크롤이 생깁니다. 가로 860px 이하로 통일하세요.

모바일 퍼스트 설계 원칙

원칙 1: 세로 스크롤에 최적화

모바일 사용자는 엄지손가락으로 세로 스크롤만 합니다. 좌우 슬라이드나 탭 전환은 최소화하세요.

원칙 2: 한 화면에 하나의 메시지

모바일 화면은 작습니다. 한 스크롤 구간에 하나의 핵심 메시지만 전달하세요. 정보가 밀집되면 아무것도 읽히지 않습니다.

원칙 3: 터치 친화적 버튼

CTA 버튼의 최소 크기는 44x44px입니다. 버튼 간 간격도 충분히 두어 오탭(잘못 누름)을 방지하세요.

원칙 4: 로딩 속도 3초 이내

모바일 사용자의 53%가 3초 이상 걸리면 이탈합니다. 이미지 압축, 지연 로딩(Lazy Loading)을 반드시 적용하세요.

실전 체크리스트

  • ✅ 모바일에서 직접 상세페이지를 확인했는가?
  • ✅ 이미지 내 텍스트가 모바일에서 읽히는가?
  • ✅ CTA 버튼이 엄지로 쉽게 누를 수 있는가?
  • ✅ 페이지 로딩이 3초 이내인가?
  • ✅ 가로 스크롤이 발생하지 않는가?
  • ✅ 핵심 정보가 스크롤 없이 보이는가?

마무리
